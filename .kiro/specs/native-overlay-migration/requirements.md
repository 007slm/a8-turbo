# 需求文档

## 简介

本文档规定了将 A8Wisper 悬浮窗动画系统从 Tauri webview 渲染迁移到纯原生 Python 渲染的需求。当前实现使用 Tauri 来显示录音状态的小悬浮窗，这增加了不必要的复杂性和打包体积。主要的 Tauri 配置界面将保持不变。此次迁移旨在实现更好的性能、更小的分发体积和简化的悬浮窗架构，同时保持所有现有的视觉效果。

## 术语表

- **悬浮窗**: 在语音输入期间显示录音状态和动画的小型浮动UI元素
- **Tauri主界面**: 使用 Tauri 构建的主配置窗口（保持不变）
- **原生渲染器**: 用于悬浮窗动画的轻量级 Python 渲染系统
- **动画引擎**: 负责视觉效果的系统，如涟漪、旋转器和脉冲效果
- **状态管理器**: 处理悬浮窗可见性和状态转换的组件
- **打包体积**: 打包应用程序分发包的总大小
- **Tkinter悬浮窗**: 用于最小化悬浮窗渲染的内置 Python GUI 工具包

## 需求

### 需求 1: 移除悬浮窗的 Tauri 依赖

**用户故事:** 作为开发者，我希望从悬浮窗系统中消除 Tauri 依赖，以便应用程序具有更小的打包体积和降低的复杂性。

#### 验收标准

1. 当应用程序构建时，系统不应包含任何用于悬浮窗功能的 Tauri 运行时文件
2. 当应用程序启动时，系统不应为悬浮窗功能初始化任何 webview 组件
3. 系统应移除与悬浮窗相关的所有 Tauri 配置文件
4. 系统应移除所有 TypeScript/React 悬浮窗组件
5. 与当前基于 Tauri 的实现相比，打包体积应减少至少 20MB

### 需求 2: 实现原生悬浮窗渲染

**用户故事:** 作为用户，我希望悬浮窗动画能够原生渲染，以便获得更好的性能和系统集成。

#### 验收标准

1. 原生渲染器应创建一个无边框、始终置顶的悬浮窗
2. 悬浮窗应支持透明背景渲染
3. 悬浮窗应定位在主显示器的底部中央
4. 悬浮窗应根据屏幕分辨率和 DPI 缩放自动调整位置
5. 悬浮窗应保持 180x44 像素的固定大小（逻辑单位）

### 需求 3: 保持动画效果

**用户故事:** 作为用户，我希望所有现有的悬浮窗动画都能完全相同地工作，以便用户体验保持一致。

#### 验收标准

1. 当状态为"RECORDING"时，动画引擎应显示扩展的红色涟漪效果
2. 当状态为"RECOGNIZING"时，动画引擎应显示蓝色轨道点效果
3. 当状态为"POLISHING"时，动画引擎应显示紫色脉冲效果
4. 当状态为"PROCESSING"时，动画引擎应显示白色旋转器
5. 动画引擎应保持 30 FPS 的动画流畅度
6. 动画引擎应在录音期间支持基于音频电平的视觉反馈

### 需求 4: 保持状态管理兼容性

**用户故事:** 作为开发者，我希望悬浮窗状态管理与现有代码保持兼容，以便对主应用程序的更改最小化。

#### 验收标准

1. 状态管理器应接受相同的状态值："IDLE"、"RECORDING"、"RECOGNIZING"、"POLISHING"、"PROCESSING"
2. 当状态为"IDLE"时，悬浮窗应隐藏
3. 当状态更改为任何非空闲值时，悬浮窗应变为可见
4. 状态管理器应在录音期间支持实时音频电平更新
5. 状态管理器应与现有的基于 WebSocket 的通信系统集成

### 需求 5: 优化性能和资源使用

**用户故事:** 作为用户，我希望悬浮窗使用最少的系统资源，以便不影响其他应用程序的性能。

#### 验收标准

1. 悬浮窗应在可用时使用硬件加速渲染
2. 动画引擎在活动动画期间的 CPU 使用率应限制在 2% 以下
3. 悬浮窗在运行期间应使用少于 5MB 的 RAM
4. 系统应在 100ms 内启动悬浮窗
5. 当悬浮窗不可见时，动画引擎应暂停动画

### 需求 6: 确保跨平台兼容性

**用户故事:** 作为开发者，我希望原生悬浮窗在不同的 Windows 版本上一致工作，以便所有用户都有相同的体验。

#### 验收标准

1. 悬浮窗应在 Windows 10 和 Windows 11 上工作
2. 悬浮窗应正确处理高 DPI 显示器
3. 悬浮窗应遵守 Windows 辅助功能设置
4. 悬浮窗应与多显示器配置兼容
5. 系统应在运行时优雅地处理显示分辨率更改

### 需求 7: 保持开发工作流程

**用户故事:** 作为开发者，我希望开发和构建过程得到简化，以便维护更容易。

#### 验收标准

1. 系统应仅使用 Python 依赖项进行悬浮窗功能
2. 构建过程不应要求 Node.js 或 Rust 工具链用于悬浮窗组件
3. 系统应在开发期间支持悬浮窗更改的热重载
4. 系统应为悬浮窗初始化失败提供清晰的错误消息
5. 系统应与现有配置文件保持向后兼容性
