# OJP UI 测试功能使用指南

## 概述

OJP UI 提供了完整的系统测试功能，通过直观的 Web 界面来测试 OJP 服务器的各种功能和连接状态。

## 功能特性

### 1. 测试类型

- **gRPC 连接测试**: 验证与 OJP gRPC 服务器的连接状态
- **gRPC 健康检查**: 检查 gRPC 服务的健康状态
- **OpenTelemetry 连接测试**: 测试 OpenTelemetry 遥测数据收集服务
- **数据库连接测试**: 验证数据库连接池状态
- **缓存功能测试**: 测试缓存系统的读写功能
- **性能测试**: 执行性能基准测试

### 2. 测试统计

- 总测试次数统计
- 成功/失败次数统计
- 成功率百分比显示
- 实时进度条显示

### 3. 测试历史

- 保存所有测试执行记录
- 显示测试状态、结果、耗时等信息
- 支持重新运行单个测试
- 支持清空测试历史

### 4. 配置管理

- 可配置 gRPC 服务器地址和端口
- 可配置 OpenTelemetry 服务器地址和端口
- 可配置测试超时时间
- 配置实时生效

## 使用方法

### 1. 启动测试界面

1. 启动 OJP UI 应用
2. 在左侧菜单中点击"系统测试"
3. 进入测试功能界面

### 2. 配置测试参数

1. 点击"测试配置"按钮
2. 设置 gRPC 服务器地址和端口（默认：localhost:9090）
3. 设置 OpenTelemetry 服务器地址和端口（默认：localhost:4317）
4. 设置测试超时时间（默认：5000ms）
5. 点击"确定"保存配置

### 3. 运行测试

#### 方法一：快速测试
- 点击对应的测试卡片进行单个测试
- 适合快速验证特定功能

#### 方法二：批量测试
1. 在"选择测试项目"中选择要运行的测试
2. 点击"运行所有测试"按钮
3. 等待所有测试完成

### 4. 查看结果

- 测试结果实时显示在界面上
- 成功测试显示绿色提示
- 失败测试显示红色错误信息
- 详细结果可在测试历史中查看

## API 接口

测试功能通过以下 API 接口与后端通信：

### gRPC 相关
- `POST /api/grpc/test-connection` - 测试 gRPC 连接
- `POST /api/grpc/health-check` - gRPC 健康检查

### OpenTelemetry 相关
- `POST /api/otel/test-connection` - 测试 OpenTelemetry 连接

### 数据库相关
- `POST /api/database/test-connection` - 测试数据库连接

### 缓存相关
- `POST /api/cache/test-functionality` - 测试缓存功能

### 性能相关
- `POST /api/performance/test` - 执行性能测试

## 技术实现

### 前端架构
- 使用 React 18 + Ant Design 5.x 构建 UI
- 使用 React Query 管理状态和缓存
- 使用 CSS3 实现响应式设计

### 通信方式
- 通过 HTTP 代理处理 gRPC 请求
- 使用 RESTful API 进行数据交换
- 支持实时状态更新和错误处理

### 代理配置
Vite 开发服务器配置了以下代理规则：

```javascript
proxy: {
  '/api/grpc': { target: 'http://localhost:8010' },

  '/api/database': { target: 'http://localhost:8010' },
  '/api/cache': { target: 'http://localhost:8010' },
  '/api/performance': { target: 'http://localhost:8010' }
}
```

## 故障排除

### 常见问题

1. **gRPC 连接失败**
   - 检查 gRPC 服务器是否启动
   - 验证端口配置是否正确
   - 确认防火墙设置

2. **OpenTelemetry 连接失败**
   - 检查 OpenTelemetry 服务是否运行
   - 验证端口配置
   - 确认网络连接

3. **数据库连接失败**
   - 检查数据库服务状态
   - 验证连接池配置
   - 确认数据库权限

4. **缓存功能异常**
   - 检查缓存服务状态
   - 验证内存使用情况
   - 确认缓存配置

### 调试方法

1. 打开浏览器开发者工具
2. 查看 Network 标签页中的请求
3. 检查 Console 标签页中的错误信息
4. 查看测试历史中的详细错误信息

## 最佳实践

1. **定期测试**: 建议定期运行测试以确保系统健康
2. **配置管理**: 根据实际环境调整测试配置
3. **结果分析**: 关注测试历史中的趋势和模式
4. **性能监控**: 定期执行性能测试以监控系统性能

## 扩展功能

测试功能支持以下扩展：

1. **自定义测试**: 可以添加新的测试类型
2. **测试报告**: 生成详细的测试报告
3. **告警功能**: 测试失败时发送告警
4. **自动化测试**: 支持定时自动测试
