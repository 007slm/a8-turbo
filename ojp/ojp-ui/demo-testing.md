# OJP UI 测试功能演示

## 快速开始

### 1. 启动应用

```bash
cd ojp-feature/ojp-ui
npm install
npm run dev
```

应用将在 `http://localhost:5173` 启动。

### 2. 访问测试功能

1. 打开浏览器访问 `http://localhost:5173`
2. 在左侧菜单中点击"系统测试"
3. 进入测试功能界面

### 3. 功能演示

#### 测试统计面板
- 查看总测试次数、成功次数、失败次数
- 实时显示成功率百分比
- 进度条可视化显示

#### 测试控制面板
- 选择要运行的测试项目
- 点击"运行所有测试"执行批量测试
- 点击"测试配置"调整参数
- 点击"清空历史"清理测试记录

#### 快速测试卡片
- 点击单个测试卡片进行快速测试
- 实时显示测试状态和结果
- 支持并发测试

#### 测试历史记录
- 查看所有测试执行记录
- 显示测试状态、结果、耗时
- 支持重新运行单个测试
- 分页显示和搜索功能

## 测试类型说明

### 1. gRPC 连接测试
- **目的**: 验证与 OJP gRPC 服务器的连接状态
- **测试内容**: 网络连接、端口可达性
- **预期结果**: 连接成功或失败

### 2. gRPC 健康检查
- **目的**: 检查 gRPC 服务的健康状态
- **测试内容**: 服务状态、响应时间
- **预期结果**: SERVING 或 NOT_SERVING

### 3. OpenTelemetry 连接测试
- **目的**: 测试 OpenTelemetry 遥测数据收集服务
- **测试内容**: 连接状态、协议支持
- **预期结果**: 连接成功或失败

### 4. 数据库连接测试
- **目的**: 验证数据库连接池状态
- **测试内容**: 连接池健康、连接数量
- **预期结果**: 连接正常或异常

### 5. 缓存功能测试
- **目的**: 测试缓存系统的读写功能
- **测试内容**: 缓存操作、命中率
- **预期结果**: 功能正常或异常

### 6. 性能测试
- **目的**: 执行性能基准测试
- **测试内容**: 响应时间、吞吐量、错误率
- **预期结果**: 性能指标达标或未达标

## 配置示例

### 默认配置
```javascript
{
  grpcHost: 'localhost',
  grpcPort: 9090,
  otelHost: 'localhost',
  otelPort: 4317,
  timeout: 5000
}
```

### 自定义配置
```javascript
{
  grpcHost: '192.168.1.100',
  grpcPort: 9090,
  otelHost: '192.168.1.101',
  otelPort: 4317,
  timeout: 10000
}
```

## 故障排除

### 常见问题

1. **测试失败率高**
   - 检查服务器状态
   - 验证网络连接
   - 调整超时时间

2. **响应时间过长**
   - 检查网络延迟
   - 优化服务器配置
   - 调整并发设置

3. **连接被拒绝**
   - 检查端口是否开放
   - 验证防火墙设置
   - 确认服务是否启动

### 调试步骤

1. 打开浏览器开发者工具
2. 查看 Network 标签页
3. 检查请求和响应
4. 查看 Console 错误信息
5. 分析测试历史记录

## 扩展功能

### 1. 自定义测试
可以添加新的测试类型：
```javascript
case TEST_TYPES.CUSTOM_TEST:
  return await grpcClient.customTest()
```

### 2. 测试报告
生成详细的测试报告：
- 测试覆盖率
- 性能指标
- 错误分析

### 3. 告警功能
测试失败时发送告警：
- 邮件通知
- 短信提醒
- 钉钉/企业微信通知

### 4. 自动化测试
支持定时自动测试：
- 定时执行
- 条件触发
- 结果通知
