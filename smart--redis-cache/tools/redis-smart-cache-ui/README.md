# Redis Smart Cache UI

一个基于React + Ant Design的现代化Redis智能缓存管理界面，提供直观的缓存规则配置和监控功能。

## ✨ 功能特性

### 🎯 核心功能
- **Redis连接管理** - 支持配置Redis主机、端口和应用名称
- **仪表板概览** - 显示缓存命中率、查询统计等关键指标
- **查询管理** - 查看所有SQL查询，支持排序、筛选和批量设置缓存
- **规则管理** - 创建、编辑、删除各种类型的缓存规则
- **表管理** - 管理数据库表，设置表级缓存策略

### 🔧 缓存规则类型
- **表匹配** - 精确匹配指定的表集合
- **表任意匹配** - 匹配包含任意指定表的查询
- **表全部匹配** - 匹配包含所有指定表的查询
- **查询ID匹配** - 根据查询ID精确匹配
- **正则表达式** - 使用正则表达式匹配SQL
- **任意匹配** - 匹配所有查询（默认规则）

### 📊 监控指标
- 缓存命中率和覆盖率
- 查询性能统计
- 访问频率分析
- 实时性能监控

## 🚀 快速开始

### 环境要求
- Node.js >= 16.0.0
- npm >= 8.0.0 或 yarn >= 1.22.0

### 安装依赖
```bash
cd tools/redis-smart-cache-ui
npm install
```

### 启动开发服务器
```bash
npm run dev
```

应用将在 `http://localhost:5173` 启动

### 构建生产版本
```bash
npm run build
```

构建文件将生成在 `dist/` 目录中

## 📱 界面预览

### 主界面
- 顶部导航栏：Redis连接状态和配置
- 左侧菜单：功能模块导航
- 主内容区：各功能模块的具体内容

### 功能模块

#### 1. 仪表板
- 统计卡片：总查询数、已缓存查询、缓存命中率、平均查询时间
- 性能指标：缓存命中率和覆盖率进度条
- 最近数据：最近查询和规则列表

#### 2. 查询管理
- 查询列表：显示所有SQL查询信息
- 筛选排序：支持按多个维度筛选和排序
- 批量操作：支持批量选择查询设置缓存
- 缓存设置：为查询设置TTL缓存时间

#### 3. 规则管理
- 规则列表：显示所有缓存规则
- 规则编辑：支持创建、编辑、删除规则
- 规则类型：多种匹配策略选择
- 状态管理：启用/禁用规则

#### 4. 表管理
- 表信息：显示数据库表详情
- 性能统计：查询时间和访问频率
- 缓存配置：为表设置缓存策略
- 性能分析：性能分布可视化

## 📋 使用说明

### 1. 连接Redis
1. 点击右上角的"连接Redis"按钮
2. 输入Redis主机地址、端口和应用名称
3. 点击"连接"按钮建立连接

### 2. 查看缓存概览
- 在仪表板页面查看整体缓存状态
- 监控关键性能指标
- 了解缓存覆盖情况

### 3. 管理查询缓存
- 在查询管理页面查看所有SQL查询
- 使用筛选和排序功能找到目标查询
- 为单个或批量查询设置缓存规则

### 4. 配置缓存规则
- 在规则管理页面创建新的缓存规则
- 选择合适的规则类型和匹配条件
- 设置合适的TTL时间
- 启用或禁用规则

### 5. 管理表缓存
- 在表管理页面查看所有数据库表
- 分析表的查询性能
- 为表设置缓存策略

## 🔧 配置选项

### 环境变量
```bash
# 开发环境
VITE_API_BASE_URL=http://localhost:8080
VITE_APP_TITLE=Redis Smart Cache

# 生产环境
VITE_API_BASE_URL=https://api.example.com
VITE_APP_TITLE=Redis Smart Cache
```

### 主题配置
在 `App.jsx` 中修改 `ConfigProvider` 的 `theme` 属性：

```jsx
<ConfigProvider
  theme={{
    algorithm: theme.defaultAlgorithm,
    token: {
      colorPrimary: '#1890ff',
      borderRadius: 6,
      // 更多主题配置...
    },
  }}
>
```
